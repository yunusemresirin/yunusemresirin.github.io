10.1 Static classes in TypeScript

In Java und C# sind statische Klassen vorhanden. Diese sind Klassen, die sich in äußeren Klassen schachteln und mit dem Keyword static markiert sind. Es können nicht nur Methoden mit dem Keyword static markiert werden, sondern auch innere Klassen. Das Keyword static wird verwendet, um Methoden oder Variablen zu markieren, die nicht zur Instanz gehören, sondern zur Klasse selbst. Daher ist sind die Methoden und Variablen schon vor der Instanziierung verfügbar, die mit dem Keyword static markiert sind.

In Java werden statische Klassen verwendet, um Daten und Funktionen nicht mit Objekten zu verknüpfen, also diese unabhängig bieten zu können, da Objekte nicht ohne Klassen existieren können.

In Java finden statische Klasse ihre Verwendung bei beispielsweise Einzelklassen, wie die Math-Klasse, bei der eine Instanziierung sinnlos wäre. Grund ist, dass beispielsweise die mathematischen Konstanten wie PI oder E und die mathematischen Funktionen für jedes Objekt gleich wäre, und man daher nur Speicherplatz verschwenden würde.

In Java-/Typescript gibt es die Möglichkeit, mit Literalen flexibel Objekte zu erzeugen, ohne vorher eine Klasse definiert zu haben.

10.2 Fehler finden mit TypeScript

Ein Fehler in TypeScript tritt unter zwei Bedingungen auf:

  • Es liegt ein gewöhnlicher Programmfehler vor.
  • Es wurde falsch typisiert.
Ein Programmfehler lässt sich definieren als ein Fehlverhalten. Also eine Abweichungen von einem erwarteten Code-spezifizierten Verhalten, welches ein entsprechendes Ergebnis je nach der Konfiguration des Inputs liefern soll. Ohne die Spezifikation des Codes/Programms lässt sich das Fehlverhalten nicht auf den Fehler zurückführen. Das heißt, dass der Fehler durch probeweisen Vergleich des Verhaltens bzw. Outputs mit dem Code ermittelt werden kann.

Das Problem bei dem Code ist die Addition der booleschen Werte. Normalerweise werden Typ-bedingt nur number-Werte miteinander addiert (als auch die restlichen arithmetischen Operationen).

Das Prüfen der Gleichheit (equals) ist jedoch mit allen Typen möglich. In Java kann man mit generischen Typparametern voraussetzen, dass die Argumente der Methode vom selben Typen entsprechen. Dasselbe Prinzip ist sowohl in JavaScript als auch in TypeScript möglich. Die Korrektur sieht folgendermaßen aus: 
                
    const add : any = (x:number,y:number) : number => { return x + y };
    const equals : any = <T>(x:T,y:T) : boolean => { return x===y };

    console.log( add(1, 1) ); // Gibt 2 aus
    console.log( equals(1, 1) ); // Gibt true aus

    var x = y = 1;
    console.log( add(x, y) ) // Gibt 2 aus
    console.log( equals(x, y) ) // Gibt true aus, da die Referenz von x und y gleich ist
