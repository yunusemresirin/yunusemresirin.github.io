1.1 Fachliche Argumentation über Erfolgsprinzipien des WWW

  1. Mit welchen fachlichen Argumenten wurde das WWW-Proposal von TBL abgelehnt?
    • Die Wagheit im Konzept war nicht verständlich und wurde mit der Involvierbarkeit verwechselt.
    • Die Hypertext-Community war der Meinung, dass die Wissenschaft viel weiter vorankam und das Konzept von Tim Burners Lee zurückgeblieben ist. (Einer der Problematiken, mit denen sich das Berner-Konzept befasste, war das "Broken Link", also Verlinkungen, die jedoch auf nicht-vorhandene Seiten bzw. Daten verweisen, somit einen Fehler im Programm darstellen)
  2. Was sind die fachlichen Argumente, warum das WWW dennoch ein Erfolg wurde?
    • Broken Links waren möglich, da der Foreign-Key der Elemente ins Leere zeigen konnte, jedoch konnte dies bei sorgfältiger Codierung verhindert werden.
    • Das WWW fand seinen Erfolg in seiner Skalierbarkeit der Codes und seiner Effizienz bei Verlinkungen auf Ressourcen. Außerdem war die Nicht-Lizensierung ein ausschlaggebender Punkt bei der Entwicklung von Servern und Clients.
  3. Was wäre der Preis für die garantierte Verhinderung von "Broken Links"?
    • Man benötigt referenzielle Integrität. Bei jedem Link muss der Foreign-Key geprüft werden. Der Attributwert des Fremdschlüssels muss auch bei einer Entität als Primärschlüssel existieren. Um Broken Links garantiert zu verhindern, müsste sichergestellt werden, dass dies immer geprüft wird.

1.2 HTTP

  1. Sie bekommen im Browser den HTTP Status Code 200. Was bedeutet das?
    • OK: Die Anfrage wurde erfolgreich bearbeitet und das Ergebnis der Anfrage wird in der Antwort übertragen.
  2. Sie bekommen im Browser den HTTP Status Code 301. Was hat das zu bedeuten?
    • Moved Permanently: Die angeforderte Ressource steht ab sofort unter der im „Location“-Header-Feld angegebenen Adresse bereit (auch Redirect genannt). Die alte Adresse ist nicht länger gültig. Der Nutzer wird umgeleitet.
  3. Sie bekommen im Browser den HTTP Status Code 400. Was hat das zu bedeuten? Was können Sie dagegen tun?
    • Bad Request: Die Anfrage-Nachricht war fehlerhaft aufgebaut. Dies lässt sich durch eine einfache Aktualisierung der betreffenden Seite beheben.
  4. Sie bekommen im Browser den HTTP Status Code 403. Was hat das zu bedeuten? Was können Sie dagegen tun?
    • Forbidden: Die Anfrage wurde mangels Berechtigung des Clients nicht durchgeführt, bspw. weil der authentifizierte Benutzer nicht berechtigt ist oder eine als HTTPS konfigurierte URL nur mit HTTP aufgerufen wurde. Der Benutzer muss sich vor dem Aufruf der Seite autorisieren.
  5. In einer Webanwendung benötigen Sie eine OPTIONS-Anfrage, die die Optionen des Servers vor dem eigentlichen Zugriff erfragen soll. Aus Performanzgründen soll die Abfrage jedoch cacheable sein. Wie könnten Sie dafür eine Lösung angehen?

1.3 HTML-Literatur lesen und Fragen beantworten

  1. Was ist HTML?
    • Steht für Hyper Text Markup Language
    • Standardisiertes Markup Language um Webseiten zu entwickeln
    • Beschreibt die Struktur einer Webseite in Form von aneinandergereihten Elementen, die dem Browser mitteilen, wie der Inhalt abgebildet werden soll.
  2. Wie kann man eine geschachtelte geordnete Liste der Schachtelungstiefe 3 erzeugen?
    • Mit geordneten Listen (<ol>) kann durch das Schachteln der Listenelementen realisiert.
    • <ol>
      <li> Erste Ebene 1 </li>
      <ol>
      <li> Zweite Ebene 2 </li>
      <ol>
      <li> Dritte Ebene 3 </li>
      </ol>
      </ol>
      </ol>
  3. Wie ist eine HTML-Tabelle aufgebaut?
    • Eine HTML-Tabelle besteht aus folgenden Elementen:
      • <table> : definiert den Tabellenblock
      • <td> : definiert eine Tabellenzelle
      • <tr> : definiert eine Tabellenzeile
      • <th> : definiert einen Tabellenheader
  4. Welche Konventionen sollte man bei Pfaden und Dateinamen beachten?
    • der Pfad sollte an der Namensdeklaration erkennbar sein
  5. Wie baut man in HTML ein Menü?
    • <nav>: HTML-Tag, um eine Navigationsleiste zu erzeugen
    • <ul> oder <ol>: HTML-Tag zur Erzeugung einer Liste
      • durch Integration eines <a> Tags, kann man eine improvisierte Menüleiste erzeugen
  6. Welche Attribute sollte man bei Bildern wie verwenden?
    • zur Integration von Bildern (englisch: Images) nutzt man das HTML-Tag <img&g. Wichtig ist, dass man das Attribut src beachtet, um auf das Bild am Speicherort zuzugreifen
    • Attribute wie height für die Größe und widthfür die Breite des Bildes sind ebenso relevant
